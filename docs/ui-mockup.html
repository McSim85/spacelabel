spacelabel

A label belongs to a Space's UUID, not its position — so it follows the desktop through any reorder. This board shows every surface that renders that label: the menu bar, its dropdown, the preferences window, and the on-switch HUD & corner overlay. macOS chrome is drawn to spec; the mint layer is annotation.

Menu-bar anatomy

One status item, one drawn view

The buttons row is one NSStatusItem hosting a custom CG-drawn view — never N separate items, which worsen Tahoe's Control Center hiding and notch overflow. Each segment is a pill: first letter(s) of the label, falling back to the Space number.

Buttons row

E C S D 5 T Do 3
  • Current = alpha, not color. The active Space is full opacity; the rest sit at ~40%. Color stays free for the user's own tags. One current per display.
  • Pill text is the label's leading letter(s); an unlabeled Space falls back to its number (5, 3).
  • Per-display split — a thin vertical divider groups each physical display, drawn left-to-right.
  • Scope toggle — menubar.buttons_scope: all displays (default) or active display only.

Status title & visibility

Email E C S
  • Primary mode — the active Space's label is the status item's title, updated on the debounced space-change.
  • Buttons row is opt-in via menubar.show_buttons_row (default off); title-only is the quiet default.
  • “Alive” ≠ “visible” on Tahoe. If the icon is hidden, the dropdown copy points to System Settings → Control Center / Menu Bar; one instance only.
  • Display-only by default. Clicking a pill switches Spaces only if you opt into click-to-switch (see legend).

The dropdown

Everyday editing, grouped by display

WhichSpace parity without the index bug. The all-Spaces list is grouped under a per-display header (friendly name resolved from the display UUID), the current Space checkmarked. We never loop-toggle isVisible (the Tahoe Control Center regression).

What each part does

  • Rename this Space… opens a text dialog → runs label set current.
  • Per-display section headers resolve the display UUID to a friendly name so per-display Spaces never conflate.
  • Checkmark = the current Space on its display; the swatch shows the user's color tag.
  • Mode toggles mirror the CLI mode command (Menu-bar title / On-switch HUD / Corner overlay).
  • Preferences… and Quit close the everyday surface; Quit stays stopped (it won't auto-relaunch).

Preferences

A display → its Spaces, inline-editable

A two-level NSOutlineView: each physical display is a parent row, its Spaces the children. Edit the label or color in place; the current Space is marked. Labels persist keyed by Space UUID — the display UUID is stored alongside for grouping only, never as part of the key.

  • Inline edit — double-click a label to edit (blue edit box); pick a color in the Color well. Commit on Return or focus-loss, write is atomic + flock-guarded.
  • Orphans retained by default. A deleted/recreated Space gets a new UUID; old labels collapse under an “Orphaned labels” group and survive until you Prune orphans (mirrors label prune). Never auto-deleted — cross-reboot UUID stability is still being verified.

HUD & overlay geometry

Sizes computed from the screen, never hardcoded

Font size, position, and fade are derived at runtime from the current display's geometry — orientation, point size, and Retina scale. The same formula must read well on a tall portrait panel, a wide 4K, and a single-display laptop.

Position is configurable. The HUD takes any of these nine anchors via hud.position — default center. The persistent overlay shares the same anchors via overlay.corner (default top-right).

Edge and corner anchors are inset by hud.margin / overlay.margin; center anchors ignore the margin on the centered axis. One shared helper computes every panel's origin.

# S = short side of the display, in points (handles portrait & landscape alike)
S = min(frame.width_pt, frame.height_pt)
hud_font = clamp(round(S × 0.05), 18, 64) pt   # scales, clamped so it's never silly
overlay_font = overlay.font_size (15 pt) | auto: clamp(round(S × 0.018), 12, 28)

# ONE anchor helper places both panels inside visibleFrame (AppKit bottom-left origin):
anchor_origin(vf, panel(w,h), pos, m):
  x = { left: vx+m,  center: vx+(vw−w)/2,  right: vx+vw−w−m }[pos.horizontal]
  y = { top: vy+vh−h−m,  middle: vy+(vh−h)/2,  bottom: vy+m }[pos.vertical]

hud: pos = hud.position (center), m = hud.margin (24) # e.g. top-left, bottom-center…
overlay: pos = overlay.corner (top-right), m = overlay.margin (12)
hud_fade = in 120 ms → hold hud.duration_ms (1100) → out 350 ms # one reused NSPanel

# Retina crispness is free: AppKit draws in points, backingScaleFactor handles pixels.
# Reposition every panel on NSApplicationDidChangeScreenParameters; HUD on the active display, one overlay per display.
